# Ковчег — Telegram Mini-App. План реализации

## Стек
- **Бэкенд**: Python 3.11 + FastAPI + SQLAlchemy + SQLite (одна файловая база).
- **Бот**: aiogram 3.x, webhook на ту же FastAPI-приложение (`/webhook/<secret>`).
- **Фронт**: статический HTML/CSS/JS (без сборки), подключается Telegram WebApp SDK с CDN.
- **Деплой**: бэкенд + статика — одно приложение на Fly.io (один HTTPS URL).
- **Хранилище данных**: SQLite на томе Fly.io (persistent volume), чтобы не терять данные между деплоями.

## Модели данных (SQLite)
- `User` — `telegram_id`, `username`, `first_name`, должность (`role` — например «Гражданин»), `restrictions` (текст или `null`).
- `Item` — каталог предметов (название, описание, картинка/иконка, редкость, тип: ресурс/ускоритель/декор/другое, можно ли активировать/дарить).
- `Inventory` — связь пользователь ↔ предмет ↔ количество.
- `Wallet` — `user_id`, баланс монет.
- `Transaction` — переводы (отправитель, получатель, сумма, дата).
- `Task` — задание (название, описание, награда, активно/нет, требует подтверждения админа).
- `UserTask` — пользователь ↔ задание ↔ статус (`in_progress`, `done`, `cancelled`), `progress`, `approved_by`, `approved_at`.
- `ShopProduct` — товары магазина (предмет, цена, в наличии).
- `MarketListing` — выставленные пользователями товары (продавец, предмет, цена, активна).
- `WheelSpin` — лог спинов (`user_id`, `prize`, `date`) — чтобы давать 1 спин/день.
- `News` — лента новостей (картинка, заголовок, дата).
- `Banner` — карточки карусели на главной.
- `LegalText` — конституция и законодательство (один длинный текст на каждое).

## Экраны
### Главная
1. Хедер: «{name}, Добро пожаловать!» + дата/время МСК + куб логотипа.
2. Карусель 3 баннеров 16:9 (свайп / точки).
3. Карточка «Ежедневное колесо фортуны» → открывает модал с колесом, реальная анимация вращения, дроп приза, попап «Поздравляем!».
4. Блок «Новость» — последняя новость, 7:5 картинка + текст.
5. Блок «План» — обязательный, нельзя стартовать (визуально locked).
6. Блок «Задания» — карусель по 3 на слайд. Клик по заданию → модалка с описанием, наградой, кнопками «Начать» / «Закрыть».
7. Три кнопки внизу: «Конституция» / «Законодательство» (открывают модал со скроллом) / «Телеграм канал» (`Telegram.WebApp.openTelegramLink`).

### Профиль
1. Хедер: имя, должность, ограничения + аватарка (плейсхолдер скина).
2. Инвентарь — сетка карточек, у каждой кнопки «Подарить» / «Активировать».
3. Кошелёк — баланс + «Перевести» (модалка с выбором получателя по username / TG ID и суммой) + «Получить» (показывает QR/ID для перевода).
4. План (дублирует обязательный из главной).
5. Задания — начатые, можно открыть карточку: «Закрыть» (скрыть модалку) / «Прервать» (cancel).

### Коверна
- Переключатель «Магазин» / «Рынок».
- Магазин: товары от Ковчега. Купил → списались монеты, предмет в инвентарь.
- Рынок: товары от других граждан. Сверху кнопки «Продать» (выбрать из инвентаря) и «Мои объявления» (управлять/снимать).
- Фильтры: «Все / Ресурсы / Ускорители / Декор / Другое».

## Аутентификация
Каждый запрос фронта шлёт заголовок `X-Telegram-Init-Data` с `Telegram.WebApp.initData`. Бэкенд валидирует HMAC-SHA256 по `bot_token`. При первом обращении — создаётся `User` с данными из `initData.user`.

## Админ
Telegram ID админа из env `ADMIN_IDS` (список). Команды в боте:
- `/approve <task_user_task_id>` — отметить задание выполненным, начислить награду, отправить уведомление пользователю.
- `/reject <task_user_task_id> [причина]` — отклонить, вернуть в `in_progress` или отменить.
- `/spawn <user_id> <item_code> <qty>` — добавить предмет (для тестов).

## Бот
- `/start` отправляет приветствие + reply-кнопку с `web_app` ссылкой на mini-app.
- Также настроен «menu button» через BotFather (или через `setChatMenuButton` API).
- Webhook принимает апдейты на `/webhook/<TELEGRAM_WEBHOOK_SECRET>`.
- При `approve` бот шлёт пользователю «🎉 Задание выполнено! Награда: X монет».

## Деплой
- `deploy backend` (Fly.io) с томом для SQLite (`/data/kovcheg.db`).
- После деплоя — `setWebhook` на полученный URL.
- Через BotFather — `/setmenubutton` указывает на `https://<app>.fly.dev/`.
